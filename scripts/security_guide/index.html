Verified items

These items were reviewed and are shown as verified. Each entry includes an optional verification or remediation note.

  1. System time & timezone — Verify system time and timezone; check Security/Event Viewer for time-change events.
  2. Credential & audit settings — Ensure auditing for credential validation is enabled and review failures.
  3. Removed unauthorized users — user1, user2 removed.
  4. Local accounts — Users user1 and user2 are not administrators.
  5. Password presence — User user1 has a password set.
  6. Created accounts/groups — Account user1 and group user2 created; users added to group.
  7. Password expiration — User user1 has password expiration enabled.
  8. Password length — A secure minimum password length is enforced.
  9. Account lockout — A lockout threshold exists to slow brute-force attempts.
  10. Audit — Credential validation auditing reported failures (review event logs).
  11. System time change restriction — Users are restricted from changing system time.
  12. Firewall — Firewall protection is enabled.
  13. Remote Assistance — Remote Assistance connections disabled.
  14. Removed artifacts — Affine encrypted text file removed.
  15. Removed software — Wireshark, NetStumbler, PC Cleaner removed.

Missing / commonly-missed items

The following items are commonly overlooked during hardening; verify and remediate as needed.

  • Guest account disabled
    Remediation (Windows): Set-LocalUser -Name Guest -Enabled $false
  • Password complexity enabled
    Remediation (Windows): enable complexity in Group Policy / local security policy (secpol.msc).
  • Password history / age
    Remediation: set Password History (e.g. 24), Minimum age (1 day), Maximum age (90 days) via Group Policy.

Other common issues to check

  • Remote Desktop disabled where not required (different from Remote Assistance).
  • Unnecessary shares removed (for example, C:\Users\Public).
  • Additional unauthorized software beyond those already removed.

Additional Linux notes & remediation

Actionable checks and commands for Linux systems. Run these as root or with sudo where appropriate.

  • SSH — Verify /etc/ssh/sshd_config has PermitRootLogin no and appropriate PasswordAuthentication settings.
    Verify: sudo grep -E "^PermitRootLogin|^PasswordAuthentication" /etc/ssh/sshd_config
    Remediate: update file and sudo systemctl restart sshd.
  • Null passwords — Remove nullok from /etc/pam.d/common-auth.
    Example: edit the line to remove nullok so it reads: auth [success=2 default=ignore] pam_unix.so
  • TCP SYN cookies — Verify: sysctl net.ipv4.tcp_syncookies
  • Minimum password length — Verify PAM pwquality settings (e.g. /etc/security/pwquality.conf).
  • User permissions — Check for incorrect perms on home dirs: find /home -maxdepth 2 -perm -o+w -exec ls -ld {} \;
  • Insecure passwords — Force resets for flagged accounts and enforce complexity/history.
  • UFW — Verify state: sudo ufw status verbose
  • /etc/shadow perms — Ensure permissions are restrictive: sudo chmod 640 /etc/shadow && ls -alF /etc/shadow
  • GRUB perms — Ensure /boot/grub/grub.cfg is not world-readable: ls -alF /boot/grub/grub.cfg and sudo chmod 600 /boot/grub/grub.cfg
  • nginx — If not used: sudo systemctl stop nginx && sudo systemctl disable nginx
  • Automatic updates — Verify unattended-upgrades, dnf-automatic, or similar is configured.
  • Applications updated — Confirm Firefox/Thunderbird updates via package manager logs.
  • AisleRiot — Remove if prohibited: sudo apt remove aisleriot && sudo apt autoremove -y
  • SSH root login — Verify PermitRootLogin no in /etc/ssh/sshd_config.
  • When automating, prefer POSIX tools, check for required commands, and test across target distros to avoid glibc/tooling errors.

Quick remediation commands

Small, copyable examples (run elevated / as appropriate for the OS):

secedit /export /cfg C:\temp\secpol.cfg

sudo chmod 600 /boot/grub/grub.cfg

sudo apt remove aisleriot && sudo apt autoremove -y
